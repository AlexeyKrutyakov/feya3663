## Легенда статусов

- ✅ **согласовано** — решение принято, можно опираться
- 🟡 **черновик** — есть первичное наполнение, требует уточнения
- ❓ **открытый вопрос** — ждёт решения (см. `05-open-questions.md`)
- ⬜ **пусто** — раздел ещё не затронут

## Метод работы

1. Идём блоками: один смысловой раздел за раз.
2. Каждый ответ может менять решения «выше по течению» — это отслеживается.
3. Ключевые технические выборы фиксируются как **ADR** (Architecture Decision Record): контекст → варианты → решение → последствия.
4. Прослеживаемость: требование → отражение в архитектуре → задача в плане.

## Конвенции проекта

- **Язык документов** — русский. **Код, имена сущностей, комментарии, коммиты** — английский.
- **Коммиты** — через skill `/commit` (Conventional Commits).
- **Тип приложения** — веб (b2c), серверная часть — основа, веб-интерфейс первым, адаптив под мобильные.
